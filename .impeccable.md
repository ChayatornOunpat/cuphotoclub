# CU Photo Club — Impeccable Design Context

## Design Context

### Users
Broad audience: current CU students, prospective members, alumni, collaborators, photography enthusiasts, and press. Visitors arrive with varying levels of familiarity with the club — some are insiders checking for updates, others are discovering the club for the first time. The site must welcome both. The job-to-be-done spans exploration (browsing the work), discovery (learning the club's story and history), and action (wanting to join or collaborate).

### Brand Personality
**Timeless · Refined · Storytelling**

The club is the official photography club of Chulalongkorn University — one of Thailand's most prestigious institutions. The brand should feel like it has earned its place: not trendy, not temporary, not trying too hard. Editorial in the truest sense — like a carefully edited journal of visual work, not a curated Instagram feed.

Emotional goals: quiet confidence, artistic credibility, a sense of legacy. Visitors should feel they are looking at something that matters and has mattered for a long time.

### Brand Assets
- **Primary color**: Hot pink `#E8186E` (from logo — used as "CU" wordmark color)
- **Logo mark**: Geometric camera lens icon in hot pink, tilted square form
- **Wordmark**: "CU" in pink, "PHOTOCLUB" in near-black — two-tone, bold sans-serif
- **Logo files**: `/logo/Arts_CUPhoto_logo - CU PhotoClubs.png`, `/logo/Copy of logo_cuphoto_1000.jpg`

### Aesthetic Direction
- **Reference**: tuphotoclub.com — clean, minimalist, image-forward, event-driven layout
- **Anti-references**:
  - Not a social media feed (no Instagram-style grids, no endless scroll of square crops)
  - Not a stock photography site (no thumbnail grids, no watermarks, no browse-by-tag UI)
  - Not corporate or bureaucratic (no committee-designed layouts, no stiff nav bars, no filler copy)
- **Theme**: Hybrid (decided). Mockup C is the chosen base — a full-bleed **dark** hero (deep charcoal `#0C0C0A`) that transitions via a signature pink cut-line into a **light** editorial body (`#F5F4F0`). Dark sections return for accent (History `#2C2A26`, footer). The contrast is intentional: drama up top, calm reading flow below.
- **Visual tone**: Large-format imagery, strong typographic hierarchy, intentional white space, restrained color use. The pink accent appears with purpose, not as decoration.
- **Typography**: **Spectral** (serif) for editorial headlines and pull-quotes — light weights (200–300), generous italics for emphasis. **Inter** (sans-serif) for body, nav, eyebrows, and UI labels — uppercase + wide letter-spacing for small labels.

### Content Structure
Landing page sections:
1. **Nav** — Logo left, minimal links (Gallery, Blog, History, About, Join)
2. **Hero** — Full-bleed featured photograph, large editorial headline
3. **Featured Gallery** — Recent work, non-grid, editorial layout
4. **Stories** — Latest blog posts as large editorial cards
5. **History** — Club origin and legacy, timeline or archival photo treatment
6. **About** — Short manifesto, who we are
7. **Footer** — Links, social, contact

### Design Principles
1. **Photography leads** — Images drive layout decisions. Typography and UI elements exist to frame the work, never to compete with it.
2. **Editorial hierarchy over UI widgets** — Use scale, weight, and space to create narrative flow. No cards with equal weight, no tabs, no accordions unless essential.
3. **The pink earns its place** — Brand pink (`#E8186E`) is strong. Use it as a precise accent: a word, a rule, a hover state. Never as a background flood or decorative fill.
4. **Legacy is visible** — The club has history worth honoring. Design choices — typography, density, restraint — should feel considered and lasting, not of-the-moment.
5. **Curated, not algorithmic** — Content is sequenced with intention. No feed logic. No "more like this." Each section has an editor's purpose.

## Implementation Reference

### CSS Tokens (`app/assets/css/main.css`)

```css
/* Tailwind v4 @theme — use as bg-accent, text-ink, ring-line etc. */
--color-accent:      #E8186E;
--color-accent-soft: color-mix(in srgb, #E8186E 12%, white);
--color-ink:         #1A1918;
--color-ink-soft:    #7C7A72;
--color-line:        #E2DED8;
--color-paper:       #EEEAE2;
--color-paper-soft:  #F0EDE6;

/* CSS variables — use directly in scoped CSS */
--hero-bg:   #0C0C0A;
--body-bg:   #F5F4F0;
--dark:      #1A1918;
--muted:     #7C7A72;
--subtle:    #E2DED8;
--paper:     #EEEAE2;
--accent:    #E8186E;
--font-serif: 'Spectral', serif;
--font-sans:  'Inter', sans-serif;
```

### Typography Scale

| Role | Size | Weight | Letter-spacing | Transform |
|---|---|---|---|---|
| Hero title | `clamp(4rem, 9.5vw, 11rem)` | 200 | `-0.03em` | — |
| Dashboard h1 | `clamp(3rem, 7vw, 6.5rem)` | 200 | — | — |
| Page titles | `clamp(2.8rem, 6.6vw, 7rem)` | 200 | — | — |
| Body / lead | `0.78–0.8rem` | 300 | — | — |
| Kicker / eyebrow | `0.54–0.58rem` | 400 | `0.2–0.3em` | uppercase |
| Nav / label / button | `0.52–0.62rem` | 400–500 | `0.16–0.18em` | uppercase |
| Small meta | `0.5–0.55rem` | 300 | `0.2em` | uppercase |

Serif font (`var(--font-serif)`) for display headings and pull-quotes only. Everything else uses `var(--font-sans)`.

Thai locale overrides: `letter-spacing: 0.035em !important; text-transform: none !important` on all uppercase labels — defined globally in `main.css`.

### Shape Language
**No border-radius anywhere in admin or public editorial UI.** All buttons, inputs, modals, image frames are sharp-cornered. The only exception is `rounded-md` in the public site's mobile nav drawer (Headless UI panel) and search icon hover — these are isolated Tailwind utilities on interactive targets, not design elements.

### Button Styles

**Admin** (`app/components/ui/Button.vue` — scoped CSS, no Tailwind):
```css
/* Shared */
font-family: var(--font-sans); font-size: 0.52rem;
letter-spacing: 0.16em; text-transform: uppercase;
padding: 0.55rem 1.1rem; /* md size */

/* Primary (dark fill) */
background: var(--dark); color: #F5F4F0; border: 1px solid var(--dark);
hover → background: var(--accent); border-color: var(--accent);

/* Secondary (ghost) */
background: none; color: var(--dark); border: 1px solid var(--subtle);
hover → border-color: var(--accent); color: var(--accent);

/* Danger */
background: #b0243c; color: #F5F4F0;
```

**Public site** uses `UiButton` with Tailwind token classes (`bg-accent`, `text-ink`, etc.) — these work because the `@theme` block registers them. Public buttons have `rounded-md`.

### Styling Architecture

| Layer | Approach |
|---|---|
| Public pages & components | Tailwind v4 utility classes + `@theme` tokens |
| Admin pages & components | `<style scoped>` with BEM names + `var(--*)` CSS variables |
| Global tokens | `:root` vars (for scoped CSS) + `@theme` block (for Tailwind) — both must stay in sync |

**Never use Tailwind utilities in admin scoped components** — classes like `text-ink`, `bg-paper-soft` work in public templates but admin uses raw CSS variables. The `UiButton` admin component is the exception: it is a shared component that uses only scoped CSS now.

### Admin UI Patterns
- **Header bar**: `background: var(--dark)`, nav links `font-size: 0.62rem`, `letter-spacing: 0.16em`, uppercase
- **Modals**: `UiModal` component, size `xl` for pickers, `flush` prop removes default padding
- **Tables**: `.tbl` class, `th` in uppercase tiny tracking
- **Pills / badges**: small uppercase label, `border: 1px solid var(--subtle)`, no background
- **Signature cut-line**: `2px` horizontal rule in `var(--accent)` — used between dark and light sections
